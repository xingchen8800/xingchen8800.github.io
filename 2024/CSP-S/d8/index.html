并查集

介绍

描述

并查集是一种 树形结构，动态 维护若干个 不重合 的集合的 查找 与 合并。

操作

  1. 合并(merge(x,y))：将两个集合合并成一个集合

  2. 查找(find(x))：查找元素所在的集合

    ### 用法

在一些有 个元素的集合应用问题中，先将 个元素分成 个独立的单元素集合，再按照题目的条件不断合并，其间要不断查找元素所属的集合。

特点：看似不复杂，数据量极大，用正常结构来解决超时超空间

在编码中，我们通常使用一个father[]来存储元素所属于的集合中最高的父节点（祖宗）。初始时，每个节点的父节点（father[i]）为自己（i）:

然后按照题目要求合并，即若将i和j合并，则father[find(j)] = find(i); //j的祖宗的父亲是i的祖宗：

路径压缩 ：如果不进行 超级加辈 路径压缩，并查集很可能退化成一段 链表，这样查找的时间复杂度超高。在查找时可以进行路径压缩，让一个集合中的子节点直接指向它们的祖宗，即return father[x] == x ? x : father[x] = find(father[x]);
